3. What is Adcelerate?

Adcelerate is a unified performance marketing platform built to replace the fragmented stack of tools Codeway's marketing teams previously used. Instead of jumping between ad managers, spreadsheets, Dropbox, and third-party tools, everything lives in one place.

The core problem it solves: Marketing teams waste time and money because their workflows are fragmented. Creative management, campaign launching, automation, and analytics live in different tools with no shared data layer. Slow reaction times cost real ad budget.

7. Product Principles

These principles guide every product decision:

  • We are user zero. If it wouldn't help Codeway's own marketing team, it shouldn't exist.
  • Design-led for performance. Powerful tools must also be easy to use.
  • Precision over velocity. Mistakes cost real ad budget. We don't ship incomplete features.
  • Data over intuition. Every feature needs traceable success metrics.
  • Fail fast, iterate with intention. Validate hypotheses quickly, pivot when wrong.
  • We win only when the user wins. Success = positive ROAS and less time wasted, not features shipped.

Rules Automation Live

Automated rules engine that evaluates campaign performance against configurable conditions and executes budget, pause, and copy actions.

Overview

Marketing teams managing paid user acquisition campaigns spend significant manual effort monitoring performance metrics and making repetitive budget and campaign-status decisions across ad platforms. Late reactions to underperforming creatives waste ad spend; slow scaling of winners leaves revenue on the table.

Rules Automation lets performance marketers define automated rules that continuously evaluate campaign performance data against configurable conditions and execute actions — such as adjusting budgets, pausing campaigns, or duplicating high performers — without manual intervention. Optionally, an AI-powered mode lets users describe their optimization intent in natural language, and a large language model makes per-target decisions on their behalf.

4. Feature Deep Dives

Rule Types & Conditions — how rules decide

Two evaluation modes:

  • Deterministic rule-based: conditions with grouped AND/OR logic, metrics, dynamic thresholds, and configurable time windows.
  • AI/LLM mode: users describe optimization intent in natural language; a large language model makes per-target decisions on their behalf.

Targeting — what rules act on

Rules can target different levels of the ad hierarchy: Campaign, Adset, Ad, or Creative. Supports both manual static selection and dynamic name-matching filters that automatically pick up new targets as campaigns are created.

Actions & Budget Management — what rules do

When conditions are met, rules can:

  • Increase or decrease budgets (by percentage or fixed amount)
  • Pause campaigns, ad sets, or ads
  • Copy high-performing campaigns
  • Exclude countries from targeting

Budget safety limits with auto-pause guards prevent runaway spend from automation errors.

Scheduling & Execution — when and how rules run

Rules evaluate on configurable schedules: every 6 hours, daily, or weekly. Cooldown mechanics prevent the same action from firing repeatedly. A dry-run / test mode lets users safely validate a rule before it takes live actions.

Slack Notifications — alerting system

Per-rule Slack integration with configurable channels and multiple notification types: action executed, evaluation summary, budget limit hit, and LLM reasoning. Notifications include clickable Meta Ads Manager links for instant context.

Automation Templates

A template gallery in the UI allows users to share and reuse rule configurations. Any drafted rule can be saved as a template with a name and description. Other team members can browse the gallery and instantiate rules from templates.

Adcelerate Onboarding › Creative Pipeline

Creative Pipeline Live

Structured system managing ad creatives from upload through testing, scaling, and long-term use across Meta and TikTok.

Overview

The Creative Pipeline is a structured system that manages ad creatives from first upload through testing, scaling, and long-term use. It provides a clear path that every creative follows so users always know which creatives are untested, which are in a test, which won, and which are running in boost or main campaigns.

Instead of tracking creatives in spreadsheets or memory, the pipeline:

  • Organizes creatives by stage: see what is ready to test, what is in a purchase test, and what is in boost or main.
  • Tracks performance metrics (CPP, CPI, Spend, ROAS) per campaign, updated hourly.
  • Keeps a full history of every move (archive, restore, create test, move to winner, move to kill, etc.) with timestamp and user.
  • Supports Meta and TikTok separately: each creative has its own pipeline state per platform.

Two Independent Pipelines: Meta and TikTok

Each creative has two independent pipelines: one for Meta and one for TikTok.

  • A creative can be in "Purchase Test" on Meta and "No Test" on TikTok at the same time.
  • Archive and restore actions are per platform: removing a creative from one platform's pipeline does not affect the other.
  • Every stage transition, campaign creation, and metric update is scoped to a single platform.

Pipeline Stages

A creative is always in exactly one stage per platform:

StageMeaning
No TestIn the pipeline but not yet in a purchase test. Ready to be added to a test.
Purchase TestCurrently in an active purchase test campaign (competing with other creatives).
WinnerPurchase test ended and this creative was marked as the winner. Ready for boost.
LoserPurchase test ended and this creative was marked as a loser.
BoostIn a boost campaign (short scale test before main).
MainIn at least one main campaign. A creative can have multiple main campaigns over time.
KillWas in boost but was marked to kill (underperforming). Shown as "Boost Kills".
ArchivedRemoved from the active pipeline. Can be restored back to No Test.

Stage Flow

No Test → Purchase Test → Winner → Boost → Main
                        → Loser
                                 → Boost → Kill

Archive and restore only from No Test. Each platform tracks stage independently.

Lifecycle Summary

  1. Creatives enter the pipeline: synced from Dropbox via Sync Creatives. They start in No Test for Meta and/or TikTok.
  2. Run purchase tests: drag creatives into groups, launch, and let them compete.
  3. Decide winners and losers: stop the test and use metrics to inform the decision.
  4. Scale winners: launch boost, then optionally move to main. Kill underperformers from boost.
  5. Metrics stay up to date: refreshed every hour.
  6. Archive creatives that don't need testing: restore from Creative Archive to reuse later.

Core Workflows

Purchase Test

  1. Drag creatives from No Test into groups on the test launch area.
  2. Click LAUNCH: test creation requests are sent in parallel across all groups for faster execution.
  3. Let the test run and accumulate data.
  4. Stop the purchase test from the Active Purchase Tests column.
  5. In the stop dialog, review Spend, ROAS, CPI, and CPP for each creative.
  6. Assign each creative as Winner or Loser.
Error handling during test creation: If a creative encounters an error, it stays in No Test and the campaign is turned off automatically. A visual indicator and tooltip explain the issue and direct the user to Meta Ads Manager to enable it manually.

Scaling: Winner → Boost → Main

  • From the Purchase Test Winners column, choose Launch boost campaign to move creatives to Boost.
  • From Boost: Move to main to enter the Main stage with at least one main campaign. Additional main campaigns can be added over time.
  • From Boost: Stop boost → Move to kill for underperforming creatives.

Archive and Restore

ActionDescription
ArchiveMoves the creative to Archived for the selected platform(s). Available from the No Test column. If the creative has both Meta and TikTok, choose which channels to archive.
RestoreIn the Creative Archive tab, creatives are shown in Meta and TikTok columns. Use Restore to move a creative back to No Test for that platform.

Notion Integration

If Notion is configured, the creative detail dialog shows additional context under the General tab: short description of the creative and an explanation or link from Notion.

Navigation

From the left sidebar in Adcelerate:

  1. Select an App using the App dropdown at the top.
  2. Open the Creative Pipeline main page to create a new test and view the Meta and TikTok pipeline boards.
  3. Open Creative Archive to view and restore archived creatives.

Use the search box above the pipeline boards to filter creatives by name. Toggle Show Losers and Show Killed to reveal or hide those columns.

Adcelerate Onboarding › Sync Creatives

Sync Creatives Live

Automated creative distribution from Dropbox to Meta, TikTok, YouTube, Google Ads, and AppLovin with scheduled and manual sync modes.

Overview

Performance marketing teams produce creative assets (videos, images, playable HTML ads) and store them in Dropbox. Getting these files into each ad platform's asset library — Meta, TikTok, YouTube, Google Ads, and AppLovin — is tedious when done manually, especially when the same creative needs to go to multiple accounts across multiple platforms.

Sync Creatives automates this distribution. It reads files from Dropbox, uploads them to the selected platforms in parallel, tracks what has already been uploaded to avoid duplicates, and feeds new creatives into the Creative Pipeline so they are immediately available for testing and campaign creation.

The system runs automatically every 3 hours and can also be triggered manually with fine-grained control over which apps, folders, platforms, and time ranges to sync.

Provide visibility into sync resultsSlack notification adoption and error resolution rate

3. Supported Platforms

PlatformStatusNotes
Meta / FacebookFully supportedVideos and images uploaded to each Meta ad account
TikTokFully supportedVideos (9:16 only) uploaded to each TikTok advertiser
YouTubeFully supportedVideos uploaded to the app's YouTube channel
Google AdsFully supportedImages uploaded directly; videos created as assets referencing YouTube uploads
AppLovinFully supported9:16 creatives and playable HTML ads uploaded to each AppLovin account

4. Key Concepts

Dropbox Folders

Each app has two types of folders configured in Settings:

  • Creative Folders (dbxFolders): Videos and images synced to all platforms
  • Playable Folders (dbxPlayableFolders): HTML5 playable ads synced to AppLovin only

Sync Methods

Three ways to specify which creatives to sync:

  • By number of days: All files modified within the last N days
  • By specific creatives: Exact creative names (bypasses date filtering)
  • By date range: Files modified within a custom start/end date window

Automated Sync

A cron job runs every 3 hours, syncing the last 24 hours of files from all apps to Facebook, TikTok, and AppLovin.

Duplicate Detection

Before uploading, the system checks Firestore for existing platform-specific upload records. Already-uploaded files are skipped and reported as "already uploaded."

Pipeline Integration

When a creative is synced, it automatically enters the Creative Pipeline at the "No Test" stage for Meta and/or TikTok, ready for testing.

5. How Sync Relates to Other Features

Sync Creatives is the entry point of the creative workflow:

  1. Sync Creatives uploads files from Dropbox to ad platforms
  2. Synced creatives enter the Creative Pipeline at "No Test" stage
  3. The Creative Feed (used by Copy Ads and Campaign Creation) pulls from synced creatives
  4. Copy Ads places synced creatives into specific ad sets
Rule of thumb: Sync first, then use creatives in campaigns.

Copy Ads Live

Bulk ad duplication tool that copies creatives into multiple ad sets across Meta and TikTok in a single operation.

Overview

Performance marketers frequently need to place the same video creatives into multiple ad sets or ad groups across Meta and TikTok. Doing this manually through each platform's ads manager is slow, error-prone, and does not scale when managing dozens of ad sets.

Copy Ads lets users select creatives from a creative feed and copy them into one or more ad sets in a single operation. The system uses an existing ad as a template, swaps in the new video, handles duplicate detection, and reports per-ad results with retry capabilities.

PlatformStatusNotes

TikTokFully supportedRegular ads, ACO (Smart Creative), and Upgraded Smart+

4. Key Concepts

Template Ad

Every target ad set must contain at least one ad. Copy Ads picks the first active ad and uses it as a template — only the video (and name) change. All other configuration (copy text, links, CTAs, tracking) is inherited from the template.

Creative Feed

The creative feed component lets users browse and select creatives by time period or name search. It is platform-aware and shows the correct creatives for Meta or TikTok. Supports bulk select/deselect.

Duplicate Detection

Before creating an ad, the system checks if an ad with the same creative name already exists in the target. Duplicates are skipped and reported, making re-runs safe.

Ad Naming Convention

On Meta, ad names follow the pattern {creativeName}__{DD/MM/YYYY} with the date appended automatically. On TikTok, the creative name is used as-is.

5. Wizard Flow

A four-step wizard guides the user through the copy operation:

  1. App & Account — select the app and ad account
  2. Ad Sets — select one or more target ad sets
  3. Creatives — browse the creative feed and select videos to copy
  4. Review & Copy — confirm the selection and submit

A selection summary sidebar is visible throughout to keep track of what's been selected.

Adcelerate Onboarding › Performance Tracking

Performance Tracking Live

BigQuery-powered analytics dashboard that unifies ad spend and attribution data across Meta, TikTok, Google Ads, AppLovin, and Snapchat with hourly ingestion, custom queries, and AI-generated newsletters.

Overview

Performance marketers need a unified view of advertising performance across multiple platforms. Checking each platform's ads manager individually is slow, inconsistent, and makes cross-platform comparison impossible. Attribution data lives in a separate system (Adjust), adding another layer of complexity.

Performance Tracking solves this by ingesting cost data from Meta, TikTok, Google Ads, AppLovin, and Snapchat every hour, joining it with Adjust attribution data (installs, purchases, revenue) in BigQuery, and exposing it through a custom query builder. Users can slice data by any combination of dimensions, apply filters, save queries for reuse, and create automation rules directly from results.

2. Goals & Success Metrics

GoalSuccess Metric
Provide a single source of truth for ad performanceReduction in time spent switching between platform dashboards

IP Shield In Dev

Two-part initiative automating IP infringement reporting and implementing technical video identification layers to protect Codeway creative assets.

This feature is currently under active development. Some capabilities described here may not yet be fully live in production.

Overview

The IP Shield feature is a two-part strategic initiative for Adcelerate. It provides a robust defense against ad jackers by automating the legal reporting process and implementing technical video identification layers. This ensures Codeway creative assets remain protected and easily attributable.

Part 1: IP Infringement Automated Reporting

Removes manual friction from IP enforcement by interfacing with the Meta IP Reporting API.

The Problem — Current Manual Process (5 Steps)

  1. Discovery: Find infringing ads via Meta Ads Library
  2. Report: Manually fill the IP report form for each ad (~15 min per report)
  3. Documentation: Log the report in Google Sheets
  4. Email loop: Respond to Meta follow-up emails with additional information
  5. Escalation: Submit page takedown requests for repeat offenders

Total time per report: approximately 20 minutes. Volume is difficult to scale manually.

The Solution — Automated Workflow

  • One-click report generation via the Meta IP Reporting API
  • Automated Google Sheets documentation of every report submitted
  • Page takedown escalation for repeat offenders

Success OKRs

  • Reduce average time per Meta report from 20 minutes to under 3 minutes
  • Increase total number of reports opened by at least 50% during the first month of use

Part 2: Creative Ownership Attribution System

A progressive three-phase system for proving video ownership. Each phase escalates technical sophistication only when the previous is insufficient.

PhaseMethodDetects
Phase 1Metadata taggingBaseline attribution — embeds ownership info in video metadata
Phase 2SHA-256 hash-based matchingExact copies — binary-identical files
Phase 3Invisible DCT watermarkingModified copies — re-encoded, cropped, or filtered videos

DCT Watermarking (Phase 3)

Discrete Cosine Transform (DCT) watermarking embeds an invisible pattern in the video that survives re-encoding, color changes, and minor cropping. This is the most technically sophisticated phase, used when hash matching is insufficient.

Success OKRs

  • Achieve 99% detection accuracy for exact video copies using hash-based matching
  • Detect at least 80% of modified or re-encoded videos that previously bypassed hash-based detection

Adcelerate Manifesto

Adcelerate's vision, market positioning, product principles, and maturity framework for the unified performance marketing platform.

Current Situation

Adcelerate operates in the rapidly expanding performance marketing market where global digital ad spend continues aggressive growth on platforms such as Meta, Google, and TikTok. As privacy regulations and AI-powered algorithms fundamentally change how ads are served, the complexity of managing profitable campaigns has reached an all-time high.

The current market is defined by extreme tool fragmentation. Marketing teams are forced to jump between disconnected applications for creative management, tracking, manual campaign launching, and third-party automation. This fragmented workflow leads to data silos, high operational overhead, and leaky budgets where inefficient spend goes unnoticed for days.

Adcelerate is uniquely positioned to disrupt this status quo. Our vision is to provide a single application for the entire performance marketing lifecycle. By consolidating in a unified value proposition, we help apps achieve three primary goals:

  • Reduced latency to launch: Moving from creative concept to live campaign in days rather than weeks.
  • Elimination of budget waste: Using real-time cross-platform automation to kill underperforming spend instantly.
  • Scaled insights: Bridging the gap between asset production, campaign management, and performance data.

Market Drivers and Positioning

We are benefiting from significant market tailwinds: the rise of short-form video, the shift toward server-side tracking, and the urgent need for AI-driven optimization that outpaces manual human management.

A critical dynamic reshaping the market is creative iteration speed. Creative is no longer just an asset management problem — it is the main optimization loop. Winning apps are defined by how fast they can learn from creative signals and iterate. As algorithms prioritize new creative variations, apps face rapid creative fatigue. Without a structured system for creative iteration and learning, even strong campaigns saturate quickly.

The Unified Value Proposition

By owning the entire marketing cycle, Adcelerate provides a level of visibility and control that point solutions cannot match. We do not just provide features — we provide a complete operating system for performance marketing.

StageWhat it covers
1. StrategyBuild media plans and forecast performance based on historical data before launching a single ad.
2. Creative OpsCreation, management, and syncing of assets. Bridge between idea and deployment.
3. ImplementationAutomate campaign setup using reusable templates. Eliminate human error in manual platform setups.

Strategic Challenges

Tension Between Breadth and Depth

Our ambitious single-application vision means we need to build feature value across a very large surface area. The challenge is to drive the right balance between breadth and depth by retaining a "seed then nurture" approach in new product areas while simultaneously improving existing tools that generate high daily usage.

Diverse Application Profiles and Global Reach

We serve a wide range of applications at different lifecycle stages — from early-stage apps looking for their first users to massive global hits managing hundreds of active campaigns. This diversity makes our business complex and creates difficulty in optimizing the UX for all app sizes.

Platform Ecosystem Dependency

A significant challenge is our reliance on the APIs and policies of external giants such as Meta, Adjust, and Apple. These platforms frequently change their data-sharing rules and technical requirements, which can impact our product functionality overnight.

Internal Value vs. B2B Features

Adcelerate currently serves as a critically important internal platform for Codeway. Our long-term goal is to transition into a standalone B2B SaaS platform. This creates a significant prioritization tension between building features that give Codeway a competitive edge and building a generalized, scalable architecture for external customers.

Product Principles

We Are User Zero and Use Our Own Product

Everything we build for Adcelerate should be a tool that our own marketing team at Codeway would use. If a feature does not provide immediate value to our internal efforts, we must challenge its existence.

We Are Design-Led for Performance

Most marketing tools are cluttered and difficult to navigate. We prioritize user experience as a core differentiator. It does not matter how powerful our automation is if it is too complex for a new marketer to master.

We Value Precision Over Velocity

In the world of paid advertising, mistakes cost real money. We do not ship incomplete or buggy features just to hit a deadline. Everything we release must be user-validated and free of errors that could lead to wasted ad spend.

We Value Data Over Intuition and Anecdotes

Every feature we build must have traceable success metrics to prove it provides value. We measure outcomes like reduced setup time or increased ROAS rather than just counting the number of launches.

We Fail Fast and Iterate with Intention

We define clear hypotheses for new features and validate them quickly. If a hypothesis is proven wrong, we pivot immediately.

We Win Only When the User Wins

Success is not defined by our code being pushed to production but by our users seeing a positive ROAS or having more time to focus on growth.

We have developed this maturity framework to help navigate the tension between breadth and depth. By categorizing features based on their integration into our internal workflows, we can make more informed prioritization decisions.

General Settings

Central app configuration hub: app identity, store URLs, Settings API, storage and caching, Snapchat integration, performance tracking, and cross-cutting fields used by multiple features.

Overview

Settings is the central configuration layer for the entire system. Every app — along with its ad accounts, Dropbox folders, performance tracking, and platform connections — is defined here. Settings are stored in Firestore and served from an in-memory cache, making reads instant and changes propagated in real time.

Each feature has its own Settings sub-spec documenting the fields it depends on. This page covers the cross-cutting settings that are shared across multiple features or don't belong to a single feature.

Important: Settings are not schema-validated on write. A typo in a field name won't cause an error but will silently break consuming features.

FieldTypeDescription
appIdstringCanonical app identifier (e.g. com.codeway.aitutor). Also used as the Firestore document ID.

When creating a new app, name, appId, and creativeCode are required.

These fields gate access to the performance tracking ecosystem. They are used by Rules Automation, Creative Pipeline, the hourly performance tracker, budget tracker, and AI newsletter.

Business Manager Mapping

FieldTypeDescription
metaAdAccountMapobjectMaps each Meta ad account ID to its Business Manager name (e.g. { "575011933319228": "BM_NAME" }). Used to select the correct API access token.
bmNamestringLegacy Business Manager name. Some older features use this instead of metaAdAccountMap.
metaAdAccountMap is critical for API authentication. A wrong mapping causes auth failures across all Meta features (Copy Ads, Campaign Creation, Budget Tracker).

Storage & Caching

App settings are stored in the Firestore collection adcelerate-apps. Each document represents one app.

Settings are loaded into an in-memory cache at server startup via a Firestore onSnapshot listener:

  • Reads are instant — getAllApps() and getAppById() serve from memory
  • Changes propagate automatically — create, update, or delete events are reflected within seconds
  • No manual refresh or server restart needed
